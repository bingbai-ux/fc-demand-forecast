# スマレジAPI調査メモ

## API種類

スマレジには複数のAPIがあることが確認できた：

1. **POS API** - POSレジ関連
2. **在庫管理 API (Inventory)** - 在庫管理関連
3. **受注管理 API** - 受注関連
4. **ウェイター API** - 飲食店向け
5. **タイムカード API** - 勤怠管理

## APIエンドポイント

| 環境 | アクセストークンAPI | プラットフォームAPI |
|:---|:---|:---|
| サンドボックス | https://id.smaregi.dev | https://api.smaregi.dev |
| 本番 | https://id.smaregi.jp | https://api.smaregi.jp |

## 現在の問題

現在使用しているのは `POS API` の `/stock` エンドポイント。
これは「在庫変動履歴」を返している可能性があり、「現在在庫」ではない。

## 解決策

**在庫管理 API (Inventory)** を使用する必要がある可能性が高い。

次のステップ：
1. 在庫管理 API のドキュメントを確認
2. 正しいエンドポイントを特定
3. バックエンドを修正
